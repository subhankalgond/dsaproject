Doubly Linked Lists (DLL): Understanding Bidirectional Navigation and Operations

Introduction

A Doubly Linked List (DLL) is a linear data structure where each node contains data and two pointers: one pointing to the next node and one to the previous node. This allows bidirectional navigation, unlike singly linked lists which only allow forward traversal.

DLLs are useful for applications requiring frequent insertions and deletions at both ends, such as implementing undo/redo features or browser history.

Key Terms

  • Node: The basic building block of a DLL. Each node has three parts: data, a pointer to the next node (next), and a pointer to the previous node (prev).
  • Head: The first node in the list. Its prev pointer is usually null.
  • Tail: The last node in the list. Its next pointer is usually null.
  • Pointers: next points to the subsequent node; prev points to the preceding node.
  • Traversal: Visiting each node in the list, either forward (using next) or backward (using prev).
  • Insertion: Adding a new node at a specific position (e.g., beginning, end, or middle).
  • Deletion: Removing a node from the list while maintaining the links.
  • Search: Finding a node with specific data by traversing the list.

Operations

DLL supports efficient operations due to bidirectional links:

  • Insertion at Beginning: Create a new node, set its next to head, head's prev to new node, and update head.
  • Insertion at End: Create a new node, set tail's next to new node, new node's prev to tail, and update tail.
  • Deletion at Beginning: Remove the head node, update the head to the next node, and free the removed node.
  • Deletion at End: Remove the tail node, update the tail to the previous node, and free the removed node.
  • Deletion (by reference): Adjust prev and next pointers of adjacent nodes and remove the target node.
  • Traversal: Start from head and follow next for forward; from tail and follow prev for backward.

Advantages

  • Bidirectional Traversal: Can traverse in both directions efficiently, unlike singly linked lists.
  • Efficient Insertions/Deletions: O(1) time for operations at known positions (e.g., head or tail), as no shifting is needed.
  • Dynamic Size: Can grow or shrink dynamically without wasting space.
  • Undo/Redo Support: Ideal for applications like text editors where backward navigation is needed.

Disadvantages

  • Extra Memory: Each node requires an extra pointer (prev), increasing memory usage compared to singly linked lists.
  • Complexity: More complex to implement and maintain due to two pointers per node.
  • Pointer Management: Risk of errors in updating pointers, potentially leading to broken links or memory leaks.
  • No Random Access: Cannot directly access elements by index; must traverse sequentially.

Examples

Below are Python examples for key operations in a DLL. Assume a Node class with data, next, and prev.

Node Struct

#include <stdio.h>
#include <stdlib.h>

typedef struct Node {
    int data;
    struct Node *next;
    struct Node *prev;
} Node;

Insertion at Beginning

Node* insert_at_beginning(Node* head, int data) {
    Node* new_node = (Node*)malloc(sizeof(Node));
    new_node->data = data;
    new_node->prev = NULL;
    new_node->next = head;
    if (head) head->prev = new_node;
    return new_node; /* New head */
}

Insertion at End

Node* insert_at_end(Node* tail, int data) {
    Node* new_node = (Node*)malloc(sizeof(Node));
    new_node->data = data;
    new_node->next = NULL;
    new_node->prev = tail;
    if (tail) tail->next = new_node;
    return new_node; /* New tail */
}

Deletion of a Node

void delete_node(Node** head_ref, Node* node) {
    if (!node) return;
    if (node->prev)
        node->prev->next = node->next;
    else
        *head_ref = node->next; /* node was head */
    if (node->next)
        node->next->prev = node->prev;
    free(node); /* Node is removed */
}

Deletion at Beginning

Node* delete_at_beginning(Node* head) {
    if (!head) return NULL;
    Node* next = head->next;
    if (next) next->prev = NULL;
    free(head);
    return next; /* New head */
}

Deletion at End

Node* delete_at_end(Node* tail) {
    if (!tail) return NULL;
    Node* prev = tail->prev;
    if (prev) prev->next = NULL;
    free(tail);
    return prev; /* New tail */
}

Forward Traversal

void traverse_forward(Node* head) {
    Node* current = head;
    while (current) {
        printf("%d\n", current->data);
        current = current->next;
    }
}

Backward Traversal

void traverse_backward(Node* tail) {
    Node* current = tail;
    while (current) {
        printf("%d\n", current->data);
        current = current->prev;
    }
}
